Tab Out — More is Less 优化方案

UI/UX Pro Max + 代码审计驱动。三大方向，18 个微优化，0 个大功能。

1

交互提速

改动小 体验飞跃

1.1 键盘全导航 — 空函数变真实功能

useKeyboard 的 ↑↓Enter 已注册但全是空函数。填上它。

行为：↑↓ 在 TabChip 之间移动焦点环 · Enter 跳转到选中的 tab · D 关闭选中 · S 保存选中 · / 聚焦搜索 · Esc 取消选中
实现：维护一个 focusedIndex state，用 ref 数组给每个 TabChip 设 focus。不需要 roving tabindex — 直接 el.focus()
时长：200ms 动画，ease-out 缓动（UX Pro Max 建议：exit 用 ease-in，enter 用 ease-out）

1.2 操作按钮半透明常驻

现在 TabChip 的关闭/保存按钮要 hover 才出现。改为常驻显示（opacity 0.4），hover 时 opacity 1。

原因：UX Pro Max — "不要仅靠 hover 展示重要操作"，touch 设备无法 hover
实现：opacity-40 group-hover:opacity-100 替代 opacity-0 group-hover:opacity-100

1.3 批量选择模式

按住 Shift + 点击可多选 TabChip，选中后底部浮出"关闭 N 个"操作条。

交互：Shift+点击 = 范围选中 · Ctrl/Cmd+点击 = 追加选中 · Esc = 取消全部
UI：选中的 chip 加 ring-2 ring-accent-blue，底部固定条显示"已选 N 个 · 关闭 · 保存"

键盘快捷键一览：

↑↓ 导航
Enter 跳转
D 关闭
S 保存
/ 搜索
Esc 取消

视觉打磨

改动中 精致感

3.1 骨架屏替代 Loading 圆点

加载时显示卡片形状的 shimmer 骨架，而非三个跳动圆点。

实现：用 Tailwind animate-pulse + 灰色矩形模拟卡片结构。2-3 个骨架卡片即可。
UX 原则：骨架屏让用户预知内容结构，感知等待时间缩短 30%+

3.2 TabChip 关闭动画 — 右滑消失

关闭 tab 时 chip 向右滑出 + 缩小，下方 chip 平滑上移填补空位。

时长：250ms · 缓动：cubic-bezier(0.4, 0, 0.2, 1)（exit 用 ease-in）
尊重：prefers-reduced-motion — 关闭动画下直接消失

3.3 Footer 数字弹跳递减

每次关闭 tab，Footer 的 tab count 数字弹跳一下（scale 1 → 1.2 → 1），颜色瞬间变红再恢复。

时长：200ms · 用 transform: scale() + color transition（不触发布局重排）

3.4 卡片 hover 微抬起

鼠标悬停卡片时上浮 2px + 阴影扩散。现在只有 shadow 变化，加一个 translateY(-2px)。

时长：200ms ease · 只用 transform + box-shadow（GPU 加速，无重排）
